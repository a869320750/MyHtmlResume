# 宝钢工程回忆录
这是我的第一份工作，刚刚毕业，来到了宝钢工程，作为刚踏入社会的第一步，这里其实还算不错，遇到了很多人很多事。

## 1、我的带教师傅

### 1.1 宋工
是个对技术比较执着的人，能看出很喜欢技术，对电脑是真的热爱。40多岁没结婚，唯一的爱好就是买笔记本，他的工位上叠了一堆的笔记本，一会儿玩玩这个一会儿玩玩那个。可惜不是科班出身的，导致基础不牢固，对软件的理解相对来说比较浅显，认为leetcode没啥用，设计模式没啥用，只有出去做项目有用。C#的那些特性有用。有一说一对面向对象这门技术确实炉火纯青。手写一个通信模块非常快，什么DAL层，BLL，Model之类的分层分的很流畅。委托泛型上来就手撕，我印象很深，因为我当时刚刚学完了C#相关的知识之后，看他写代码真的是感觉很巧妙，我当时就感叹我一辈子也写不出这样的代码。

### 1.2 汪工
是个性格比较老实本的人，五六十岁一把年纪，当年也是南京大学毕业的大佬，在宝钢工作了得有二三十年了，但是技术就停留在当时刚入职的状态了，不会用任何新的库，git工具都不认识，压缩包交付，甚至都没有交付，就是他本人去对应的钢厂，把程序跑起来，开机自启动，重启了能继续跑就完事了。优势就是确实很有经验，主要体现在定位问题上面。哪怕不怎么懂底层技术，但是遇到问题一点不慌，大脑数据库里面存了很多典型的场景，直接就能猜到具体原理。但是从他身上我看到了我在这家公司继续呆下去的状态，伤仲永说的不就是这种吗。

### 1.3 胡工
这是和我一起入职的另一个同事的带教师傅，平时也会和我交流一下。早年在华为呆过一年，还把人家代码抄到笔记本上自己保存端详。能看出相对来说更有经验，视野会更好。我当时已经有了华为的offer了，但是是外包我不打算去，他劝了一下之后，我还是决定去了，因为时间不容荒废。所以我感觉对我来说算是贵人了。我当时和他吐槽，我说我本来是学机器学习算法这些高端专业的，结果来了这里每天crud，公司把我耽误了。他的一席话让我印象很深：“你不要说公司耽误了你，你是我们里面学历最好的，你要有自己的判断，这里不适合你发展，你就要去找适合你发展的地方，而不是抱怨公司”。感觉这句话当时就把我点醒了，我后来就决定离职去华为的OD岗了。

## 2、项目开发
### 2.1 通信项目
宝钢二炼钢的一个L2项目的通信模块，我们叫L2，架构超级简单，就是启动一个socket服务端，等待L1启动socket客户端接入给我发生产数据，我把收到的数据存到本地的一个数据库里面，然后我把数据库里面的数据取出来，打包一下通过socket客户端发给L3启动的socket服务器。L2这边有其他的软件读这个数据要做一些计算之类的。这个项目当时搞了两周多，然后就去工厂里面持续的现场调试，进程跑起来了就完结了。
我这时候的技术能力非常的差劲，我连socket是啥都不知道，本科时候学的计算机网络这里基本忘光了，我是bilibili上面找了一个人C#的socket实战视频，我跟着视频一点一点敲的代码，然后结合业务把这个软件实现出来了。第一个项目来说，对我来说提升还是挺大的。

### 2.2 铁水脱硫项目
和通信项目其实是同一个业务，只不过这次我负责了大头了，整个展示界面都是我来了。然后通信这一块业务是独立的进程。架构部分就是一个基于数据库的架构，通信这一块还是一样的socket server读数据存数据，socket client发数据。然后就是一个界面了。基于winform实现的一套高度同质化的小界面。最低层是通过sql语句查表，在用户界面用一个datagrid之类的展示出来。然后会有一些需要修改的也是界面上画一下，点确定修改。总之就是典型的增删改查crud。

> 这里面我的收获以及贡献体现在几个方面：
1. 我们的界面上面需要crud的表涉及20多个，原本的架构里面，每一个表都有：
一个单独的MODEL层文件，里面是某个表的字段属性；
一个BLL层的文件，全是写死的超长sql语句字符串；
一个VIEW层界面文件，里面全都是固定编码的字段属性和类型。

这三个文件必须所有的修改都一一对应。也就是20多个三元组文件同步修改。在这里，我在出差的过程中，就做了一个基于泛型和委托的抽象化的重构。把所有的表结构字段属性都控制在model内部。BLL层我做了一个简单的基于stringBuilder的字符串组装能力，基于不同的model文件，根据我设计好的泛型函数，自动的组装出原本的增删改查sql语句。然后底层还是和以前一样的发给数据库获取数据。View层，我也是写了一个基于泛型的函数，根据输入的MODEL输出对应的datagrid表格。讲真我觉得这个东西我当时设计的还可以。

因为我们的交接材料是excel的，里面通过表格记载了和他们通信的字段名称和类型。我就发挥我读研时候的数据清洗能力，把这些文件简单处理了一下，然后编了一个小程序，把这个表格转换成C#的代码块（MODEL层）以及创建表的SQL语句脚本。这样我就可以减少很多的重复工作。

2. 完整的部署流程
我一开始去青钢的时候，厂房都没建好，我还在打磨我的软件，我需要从一个刚刚采购好的服务器上面做出一整套完整的部署。服务起来了从上电，插网线开始，然后装系统，配置基础的C#环境（汪工的做法就是装一个visual studio 2010），然后部署我们的软件，和别的部门联调。这时候的我其实对这些网络配置都弄不明白，相当于是先实践了一下，后来再看一下书就会有点收获。但是我感觉我对网络仍然不敢说是100%掌握。还是有很多概念不懂得，只是这里实践过会增深一些理解。
当时我旁边的同事就是在那里配置一个Linux环境，我们是windows server，我就感觉人家好高端，我们属事实有点菜。

## 3、技术学习
C#这里，主要是对语言本身印象很深，我当时就死命研究那个C#本质论7.0，买了一本电子书，一边画思维导图一边看，当时确实下了功夫了，虽然很多东西都忘记了，但是一些面向对象的概念深入脑海了。能看出作者确实是个大牛，写的通俗易懂，基本上每一句话每一段文字和例子我都是静下心来慢慢看的，我感觉我读研时候都没这么认真的看过书。

书看的差不多了，我也就度过了新手保护期，开始接项目了，真的接项目我还是蛮兴奋的，我当时的心态就是，我能做出贡献，贡献价值对得起我的工资就挺好的，不过我现在也基本这么想。这时候我连线程是啥都不知道，我还在那看那个C#的语法糖async await，基础都不懂上来就看这个属实是离谱了。

## 4、总结
总得来说这两年时光不算是很虚度，学到了不少，也经历了不少。遇到了这几位师傅也是各有特点。
