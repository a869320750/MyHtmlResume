# 简历去重压缩四刀总结

> 更新时间：2026-03-05
>
> 目标：让主简历 / ATS / 一页版形成统一的“可投递表达”，减少重复、提升信息密度与可读性。

## 第一刀：模板化（数据结构升级）

### 做了什么
- 为 `work` / `projects` 每条经历统一增加模板字段：
  - `id`
  - `tags`
  - `oneLiner`
  - `value3`
  - `star`（先留空模板）

### 目的
- 从“散文式描述”升级为“分层可配置内容”。
- 支持后续按岗位/场景动态选择展示层级。

### 主要文件
- `data/work.json`
- `data/projects.json`

---

## 第二刀：渲染层去重（主简历 + ATS + 导出）

### 做了什么
- 在渲染逻辑中支持 4 级展示：
  - `oneLiner`
  - `value3`
  - `star`
  - `full`
- 增加策略解析：可按 `defaultLevel / byId / byTag` 选择展示。
- 去掉 full 模式下的重复堆叠：
  - `full` 不再重复显示 `value3`，保留“oneLiner + 明细 + 可选STAR”。
- 纯文本导出（ATS导出）同步去重逻辑。

### 目的
- 同一条信息不在“摘要 + 明细”中重复出现。
- 先保证可读性，再保留信息完整度。

### 主要文件
- `scripts/resume.js`
- `scripts/ats.js`

---

## 第三刀：额度压缩（detail 模式统一瘦身）

### 做了什么
- 统一下调各 profile 的 `detail.limits`，包括：
  - `introMaxChars`
  - `workCompanies`
  - `workDetailsPerCompany`
  - `projectCount`
  - `projectWorkBullets`
  - `projectTextMaxChars`
  - `skillCount`
  - `githubCount`
- `general / developer / pm / system / architect` 的 detail 档位已统一为更紧凑的节奏。

### 目的
- 默认详细版也更接近“可投递长度”（不需要每次都切 onepage）。

### 主要文件
- `data/profiles/general.json`
- `data/profiles/developer.json`
- `data/profiles/pm.json`
- `data/profiles/system.json`
- `data/profiles/architect.json`

---

## 第四刀：策略化 detail（重点 STAR，次要降级）

### 做了什么
- 为以下 profile 的 `detail` 增加 `contentPolicy`：
  - `general`
  - `developer`
  - `pm`
  - `system`
- 策略原则：
  - 默认 `value3`
  - 关键项目/经历（按 `id/tag`）提升为 `star`
  - 次要或历史相关条目降为 `oneLiner`
- `architect` 原本已有 detail 策略，本轮保持一致。

### 目的
- 默认详细版不再“全量 full”，而是自动聚焦最有价值内容。
- 减少冗余，保持岗位信号清晰。

### 主要文件
- `data/profiles/general.json`
- `data/profiles/developer.json`
- `data/profiles/pm.json`
- `data/profiles/system.json`

---

## 附加改造：技能区可信化（并行完成）

### 做了什么
- 技能字段从 `percent` 改为：
  - `level`
  - `keywords`
  - `evidence`
- 页面与 ATS 渲染同步适配（旧字段兼容）。

### 主要文件
- `data/skills.json`
- `scripts/resume.js`
- `scripts/ats.js`
- `css/style.css`
- `css/ats.css`

---

## 当前推荐预览入口

- 通用详细 ATS：`html/ats.html?profile=general&mode=detail`
- 开发详细 ATS：`html/ats.html?profile=developer&mode=detail`
- 架构一页 ATS：`html/ats.html?profile=architect&mode=onepage`
- 主简历详细：`html/resume.html?profile=general&mode=detail`
